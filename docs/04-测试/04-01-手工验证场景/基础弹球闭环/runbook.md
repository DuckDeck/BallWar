# 基础弹球闭环手工验证场景

> 所属功能域：基础弹球闭环  
> 版本归属：V0.1-MVP | 生命周期状态：已审核

## 执行环境

- Godot 4.7 编辑器。
- 从 `res://scenes/main.tscn` 启动，使用 1080×1920 设计视图或等比窗口。
- 本文档既是手工验证规范，也是其执行绑定资产。

## 用例

### T1-MAN-01：初始可瞄准状态

- 来源层引用：L1-BW-01、L2“可瞄准”。
- manual_runbook_ref：`docs/04-测试/04-01-手工验证场景/基础弹球闭环/runbook.md#t1-man-01`
- execution_ref：`tests/t1_smoke_test.gd#_initialize`
- 前置条件：刚启动主场景。
- 操作步骤：观察场地顶部发射位和底部回收区，不进行输入。
- 期望结果：显示 Score 0（一行文字位于左侧分隔虚线正上方）、一枚生命值为 3 的数字障碍物、顶部两条斜线中央 96 像素缺口内的一颗黄绿色待发球和底部弧形回收区；没有活动球或大型发射器图标。

### T1-MAN-02：合法发射与反弹

- 来源层引用：L1-BW-01、L1-BW-02、L2“拖拽瞄准”。
- manual_runbook_ref：`docs/04-测试/04-01-手工验证场景/基础弹球闭环/runbook.md#t1-man-02`
- execution_ref：`tests/t1_smoke_test.gd#_initialize`（覆盖发射后的回合状态恢复，不替代拖拽手势观察）。
- 前置条件：处于可瞄准状态。
- 操作步骤：在顶部发射圆圈下方任意位置单击一次；再从该区域拖拽到另一处并松开，使瞄准线指向障碍区域。
- 期望结果：两种操作都不需要点中缺口；每次准备状态只出现一颗球，球从顶部两条斜线之间的缺口沿向下方向进入场地，斜向发射时可在左右边界反弹；活动球期间再次操作不产生第二颗球。

### T1-MAN-03：扣血、消除与得分

- 来源层引用：L1-BW-03、L1-BW-04。
- manual_runbook_ref：`docs/04-测试/04-01-手工验证场景/基础弹球闭环/runbook.md#t1-man-03`
- execution_ref：`tests/t1_smoke_test.gd#_initialize`
- 前置条件：障碍生命为 3。
- 操作步骤：重复调整发射方向，使球累计命中同一障碍物三次。
- 期望结果：每次命中数字减 1；第三次命中后障碍消失，Score 从 0 增至 10；前两次命中不增加分数。

### T1-MAN-04：回收与连续回合

- 来源层引用：L1-BW-05、L5“回合收束”。
- manual_runbook_ref：`docs/04-测试/04-01-手工验证场景/基础弹球闭环/runbook.md#t1-man-04`
- execution_ref：`tests/t1_smoke_test.gd#_initialize`
- 前置条件：已完成至少一次发射。
- 操作步骤：等待球触到底部浅托底曲线，观察其滚出侧墙、墙外上行并沿顶部斜线返回缺口；重复完成三次发射—回收。
- 期望结果：每次活动球抵达顶部缺口后都重新显示可瞄准状态；若已清除障碍则下一回合补充生命值为 3 的障碍；连续三回合无脚本错误。最大存活时长仅用于异常兜底。

### T1-MAN-05：重力回落

- 来源层引用：L1-BW-06、L5“反弹与命中”。
- manual_runbook_ref：`docs/04-测试/04-01-手工验证场景/基础弹球闭环/runbook.md#t1-man-05`
- execution_ref：`tests/t1_gravity_test.gd#_initialize`
- 前置条件：处于可瞄准状态。
- 操作步骤：使球在碰撞后朝上方运动，或观察一次明显的向上反弹。
- 期望结果：球的上升速度持续减小，到达最高点后转为向下运动；反弹不能将速度重置为固定向上速度。

### T1-MAN-06：运动拖尾

- 来源层引用：L1-BW-08、L2“球体运动”。
- manual_runbook_ref：`docs/04-测试/04-01-手工验证场景/基础弹球闭环/runbook.md#t1-man-06`
- execution_ref：`tests/t1_gravity_test.gd#_initialize`（自动验证路径点生成；拖尾形态需人工视觉确认）。
- 前置条件：处于可瞄准状态。
- 操作步骤：向下斜向发射球，观察其直线运动、撞击障碍物后的转向与向上反弹阶段。
- 期望结果：球移动时后方始终显示黄绿色拖尾；拖尾从远端透明且细小，向球头逐渐变宽变亮；转向和反弹后，尾迹跟随新的实际运动路径；球回收后拖尾一并消失。

### T1-MAN-07：底部托底与顶部回归

- 来源层引用：L1-BW-10、L1-BW-11、L2“托底回收表现”、L5“托底回收实现”。
- execution_ref：`tests/t1_bottom_recovery_test.gd#_verify_bottom_recovery`
- 前置条件：处于可瞄准状态。
- 操作步骤：分别让球从场地中心左、右两侧落到底部浅曲线。
- 期望结果：球首次触线后不再反弹；左侧落点向左、右侧落点向右滚动。球穿出对应侧墙后沿墙外上行，再沿顶部斜线真实移动到中央缺口；活动球到达缺口后才消失，待发球球随即显示。
