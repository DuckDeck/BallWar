# 施工蓝图：顶部蓄球的凹槽物理表现

> 上游输入：`_shared/T2-顶部蓄球轻量物理设计方案.md`、用户 2026-07-20 决策。
> 状态：已归档

## 1. 任务边界

将经典模式顶部预览替换为轻量凹槽物理，并让顺序发射等待真实入槽；不修改场内球、方块和挑战模式的回收资格。

## 2. 变更清单

### 修改

- `scripts/game_config.gd`：增加顶部预览物理的集中参数。
- `scripts/launcher.gd`：维护预览球速度、斜坡/门/球间约束，并发布入槽就绪信号。
- `scripts/ball_launch_sequence.gd`、`scripts/ball_manager.gd`：增加经典模式后续出球的入槽确认等待。
- `scripts/game_controller.gd`、`scripts/main.gd`：在管理器与发射器间中转确认信号；挑战模式走即时确认。
- 多球 L1/L2/L5/L7 与 T2 回归：记录自然动作和出球门控。

## 3. 关键实现点

- 预览球用局部二维圆形约束，不加入 Godot 碰撞世界；每帧只处理十球以内的成对推挤。
- 闭口时中心支撑凹槽；开口时允许最低球进入发射口，已占口球作为静态支撑，其余球继续受重力补位。
- `BallManager` 只等待经典模式的后续球；首球仍由玩家点击后立刻从已占据的发射口创建。

移除经典模式“基于固定补间时间即可视为补位完成”的语义，统一以预览球实际占据发射口为准。
