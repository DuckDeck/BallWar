# 轻量设计方案：T3 棋盘波次与危险线

> 状态：已归档
> 真值基线：用户 2026-07-19 明确的底部随机单行、球与数字随进程增长、全体球回收后整体上移、顶部虚线失败规则；[波次推进与危险线需求](../../../01-需求/01-03-波次推进与危险线/README.md)。

## 1. 背景与意图

游戏压力由方块自下向上堆积产生，而非 T3 旧方案中的底部危险线。多球批次完成后，棋盘推进一格；安全时补一行并提高下一回合资源，危险时冻结分数并结束。

## 2. 范围与非范围

本次：网格真值、底行随机生成、整体一格上移、危险线失败、难度增长、六种障碍形状与最终分数结算信号。

不做：奖励物、特殊球效果、结算 UI、美术动画数值、关卡编辑器。

## 3. 决策表

| 决策点 | 选定 | 放弃 | 原因 |
|---|---|---|---|
| 空间模型 | `BoardState` 保存逻辑 `(column,row)`。 | 用方块像素坐标累计位移。 | 一格上移、单行容量和触线判定可精确、可测试。 |
| 生成位置 | 每个安全回合仅在底部 `row=0` 随机生成一行。 | 在场地任意位置散点生成。 | 满足自下而上的压力和“一排最大容量”。 |
| 结算顺序 | 球全部回收 → 上移 → 危险检查 → 安全时补行/增长。 | 先补行或每颗球回收就推进。 | 避免失败回合多生成一行，也避免多球重复推进。 |
| 失败线 | 顶部虚线，方块上边缘触线即失败。 | 底部危险区或方块中心触线。 | 与用户指定的可见危险线一致。 |
| 随机性 | 本局显式种子。 | 全局随机且无法复现。 | 便于测试随机行与线上问题。 |
| 基础成长 | 安全回合提高下一批球数和底行数字范围。 | 仅通过奖励物加球。 | 符合“随游戏进行越来越多/数字越来越大”。 |
| 形状与朝向 | 六边形、五边形、正方形、等边三角形、圆形、菱形随机生成；除圆形外绕中心随机旋转，数字保持正向。 | 只用六边形或让数字随节点旋转。 | 同时满足多形状可扩展性与可读性。 |
| 数字来源 | 新行每块数值从“下一回合球数 × 1～2”中抽取，范围允许时优先不同。 | 一整行使用同一个数字。 | 数值随球量成长且单行更易辨识。 |

## 4. 状态流

`BALLS_ACTIVE → RESOLVING → (GAME_OVER | READY)`。

RESOLVING 期间，BoardController 对当前 `batch_id` 只推进一次。方块上移后一旦触顶，直接进入 GAME_OVER；安全路径才创建新底行、更新下一回合球数、恢复 READY。

## 5. 关键技术取舍

- 方块节点是棋盘状态的投影：销毁时必须释放占用，移动时以 BoardState 的新坐标刷新视觉/碰撞。
- 由于棋盘推进发生在所有球回收后，方块一格动画可以使用 Tween，不参与运行中物理。
- 球数、每行数量、横向格数和形状/数值倍率均放在 Resource；`WaveGenerator` 依据下一回合球数生成实际数字，不在 Controller 写死公式。
