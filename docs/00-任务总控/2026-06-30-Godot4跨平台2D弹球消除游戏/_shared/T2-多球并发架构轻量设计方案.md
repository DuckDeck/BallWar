# 轻量设计方案：T2 多球顺序发射架构

> 状态：已拍板并已实施
> 真值基线：用户于 2026-07-19 明确确认“球一颗一颗发射；待发球在顶部两条斜线构成的槽内；后续球按首发角度依次发射”。

## 决策

采用 `BallBatch + BallLaunchSequence + BallManager`：首球立即实例化，后续 `BallDefinition` 保留为纯数据队列，`BallManager` 每隔 `ball_launch_interval_seconds` 实例化一颗。待发视觉由 `Launcher` 根据队列绘制，不实例化 `Ball`，因此不参与碰撞。

同批球复用首球瞄准方向，`launch_spread_degrees` 固定为 0。活动球继续使用 BALL Layer 2、WORLD Mask 1，彼此不碰撞；回合完成条件升级为“待发队列和活动球集合都为空”。

## 边界

- 包含：顺序节拍、顶部槽内待发预览、同角度发射、批次完成条件、相关自动与手工验收。
- 不包含：特殊球具体效果、对象池、暂停/结算 UI、棋盘规则改动。
