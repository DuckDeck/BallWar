## 1. 任务边界

创建一个 Godot 4.7 + GDScript 的竖屏 2D 工程，提供顶部向下单球拖拽发射、边界反弹、一个数字障碍物扣血与消除得分，以及“发射—回收—再次可发射”的最小回合。

不创建多球队列、奖励效果、完整生成、暂停/计时/完整 HUD、最终美术、位图资产、导出预设或 Android TV 输入；不接入服务端、数据库、存档、插件或 Autoload。

## 3. 关键实现点

1. `Ball` 只以固定速度和 `move_and_collide` 在 `_physics_process` 移动；拿到碰撞法线后反射方向。命中障碍是同步物理动作，障碍通过类型化 signal 将“受击/销毁”上报给控制器。
2. `GameController` 是局内唯一状态拥有者，不使用 Autoload。只在 `READY` 接受发射；回收事件触发 `RESOLVING`，再延后切回 `READY`，避免同一帧重复发射。
3. 所有会影响手感的数值只存在于 `GameConfig` `.tres`；场地的视觉边界、物理边界与底部回收线使用同一组设计坐标。
4. 输入层同时识别鼠标和单指事件；从顶部发射位向下拖拽，长度不足或方向不朝下时不发射。Input Map 命名为后续 Android TV 映射预留，但 T1 不处理遥控器。
5. 场地、球和障碍均由 `_draw` 和 Godot 标准节点绘制；不读取或生成任何位图素材。
