# 施工蓝图：T2/T3 多球、棋盘波次与结算代码架构

> 上游输入：`T2-多球并发架构轻量设计方案.md`、`T3-棋盘波次与危险线轻量设计方案.md`、L1/L2/L5 对应文档
> 状态：已归档

## 1. 任务边界

本次将单球、单障碍、像素坐标回合链路改造为可运行的批次发射与逻辑棋盘链路：同帧多球、球间物理隔离、同色拖尾、统一命中、底行波次、整体上移、危险线判定和安全回合成长。

不实现爆炸/穿透/大体积球的具体效果、暂停和结算 UI、对象池、Android TV 输入或额外障碍形状；这些能力只获得数据和命中扩展入口。

## 2. 变更清单

### 新增

- `scripts/ball_definition.gd`、`ball_runtime_state.gd`、`ball_batch.gd`：球的共享定义、单实例状态与批次元数据。
- `scripts/ball_manager.gd`：创建、注册、注销活动球，并且仅在批次清空时发出完成事件。
- `scripts/hit_context.gd`、`hit_result.gd`、`hit_resolver.gd`、`ball_effect.gd`：统一命中协议与特殊球行为入口。
- `scripts/board_layout.gd`、`board_state.gd`、`wave_generator.gd`、`progression_state.gd`、`board_controller.gd`：逻辑网格、可复现底行生成、成长及一次性波次结算。
- `tests/t2_t3_architecture_test.gd`：批次、隔离、最后回收、棋盘推进和危险线回归。

### 修改

- `scripts/game_config.gd`、`resources/game_config.tres`：补充批次、调色板、棋盘、波次和成长参数。
- `scripts/ball.gd`、`scenes/ball.tscn`：消费球定义/运行状态，使用 BALL(2) → WORLD(1) 碰撞层，并经命中协议攻击障碍。
- `scripts/obstacle.gd`：以 `receive_hit` 作为唯一受击入口，记录逻辑棋盘格但不承担棋盘推进。
- `scripts/game_controller.gd`、`scenes/main.tscn`：以 `BallManager` 和 `BoardController` 编排批次与波次，加入 `GAME_OVER` 状态。
- L5/L7、项目概览和任务总控：同步真实对象边界、运行方式与自动化用例。

## 3. 关键实现点

- `GameController` 只创建 `BallBatch`、接收 `batch_finished`、提交棋盘结算；不再保存 `_active_ball` 或 `_active_obstacle`。
- `BallManager` 的活动表以批次 ID 归属球；单球回收只注销自身，表清空才发出一次完成事件。
- 棋盘以 `(column,row)` 为真值。结算顺序固定为：上移 → 危险线 → 安全时成长与底行生成 → READY。
- `Obstacle.receive_hit(HitContext)` 与 `HitResult` 防止已销毁障碍重复得分；未来球效果只能通过 `BallEffect` 改写命中上下文。

## 4. 施工切片

| 切片 | 内容 | 完成判定 |
|---|---|---|
| A | 球定义、批次和 Manager；单球改为定义驱动 | 可同帧创建多球、颜色与拖尾一致、球不碰球。 |
| B | 命中协议与障碍受击迁移 | 多球同帧命中不会重复销毁或重复计分。 |
| C | 棋盘、生成、成长和危险线 | 底行可复现生成；同一批次只推进一次，触顶进入 GAME_OVER。 |
| D | Controller/场景集成与回归 | 全部回收才结算，并覆盖至少 10 球的结构验证。 |

## 5. 历史资产退出

- 移除 `GameController` 的单球和单障碍私有引用；不保留兼容分支。
- 保留 `GameConfig` 中既有球手感字段，迁移后的默认 `BallDefinition` 继承这些基础数值，避免无关手感回归。

## 6. 验收映射

| 已拍板规则 | 落点 |
|---|---|
| 多球独立、球不碰球、同色拖尾 | `BallDefinition`、`BallRuntimeState`、`BallManager`、`Ball` 碰撞层 |
| 最后一球才收束回合 | `BallManager.batch_finished` → `GameController` |
| 底行、上移、危险线、成长 | `BoardState`、`BoardController`、`WaveGenerator`、`ProgressionState` |
| 不重复结算 | `BoardController` 批次 ID 守卫与 `Obstacle.receive_hit` |

## 7. 验证与回滚

运行既有 T1 回归和新增 `t2_t3_architecture_test.gd`。如架构回归无法定位，回滚本蓝图涉及的新增模块和 Controller/场景改造，不回滚用户此前的计时、重力、拖尾和托底回收修改。

## 8. 可开工性结论

可以开工。默认起始球数保持 1，安全回合再按配置增长；至少 10 球的并发能力以自动化回归验证，不将未拍板的特殊球玩法或结算 UI 混入本次施工。
