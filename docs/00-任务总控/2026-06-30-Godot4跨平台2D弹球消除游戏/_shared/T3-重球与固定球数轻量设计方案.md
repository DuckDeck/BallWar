# 轻量设计方案：T3 重球与固定球数

> 状态：已回补失效  
> 真值基线：用户 2026-07-20 明确决策：移除按回合和按时间自动加球；新增半径为普通球 130%、更受重力影响的重球，命中障碍时以可调 15% 概率造成 2 点伤害。

## 1. 背景与意图

球数不能再因回合或挑战模式时钟增长；本轮仅保留开局球库存。新增重球以更大体积和更高重力形成清晰差异，并以独立概率获得双倍伤害。

## 2. 范围与非范围

本次改：移除两种自动加球路径；重球定义、初始抽取、碰撞半径、重力和双倍伤害；相关自动化回归。

本次明确不改：奖励块、球库存持久化、其他特殊球、障碍形状/波次节奏、场内球之间碰撞。

## 3. 决策表

| 决策点 | 选定 | 放弃 | 为什么 |
|---|---|---|---|
| 球数来源 | 经典安全回合和挑战定时波次均不再新增球；当前批次数固定为开局数量 | 保留任一过渡自动奖励 | 防止难度/球数被隐式时间或回合驱动。 |
| 重球出现 | 新开局批次按独立、可调概率抽取重球，默认 20% | 每回合固定赠送或全部替换为重球 | 在奖励块尚未实现时让新类型可被实际体验，同时不恢复自动加球。 |
| 重量语义 | 通过 `gravity_multiplier` 提高下落加速度，默认 1.3 倍 | 引入球间质量碰撞 | 已发射球彼此不碰撞；重力是当前物理中可见且无歧义的重量表现。 |
| 双倍伤害 | 每次命中障碍独立以可调概率将伤害从 1 提升为 2，默认 15% | 永久 2 点伤害或改写障碍生命逻辑 | 保持重球的随机爆发特征，且复用既有命中契约。 |
| 体积呈现 | 场内碰撞体、球体、拖尾、瞄准扫掠和待发预览均使用 1.3 倍半径 | 仅放大绘制 | 命中范围与视觉必须一致。 |

## 4. 业务规则

| 场景 | 规则 |
|---|---|
| 开局 | 按固定开局球数创建球定义；每颗独立决定普通或重球类型。 |
| 安全回合 / 定时波次 | 只推进棋盘并按固定球数计算新行数字，不生成新球。 |
| 重球飞行 | 半径为普通球的 130%，重力倍率为配置值。 |
| 重球命中 | 每次障碍命中独立掷一次概率；成功时本次伤害为 2，否则为 1。 |
| 回收与下一批 | 回收球保留自己的类型和参数，不重新抽取。 |

## 5. 关键技术实现

- `BallDefinition` 持有类型、半径倍率、重力倍率和重球双倍伤害概率；`Ball` 在创建碰撞查询前读取它。
- `GameController` 仅在新局构造初始定义，并用可复现随机源决定重球；后续回收只复用定义。
- `GameConfig` 集中暴露重球出现率、尺寸、重力和伤害概率；`Launcher` 以定义半径绘制和计算瞄准/预览。

## 6. 影响面与回补清单

- L1：`01-02-多球并发弹射`、`01-03-波次推进与危险线`、`01-04-挑战模式`。
- L2：`mobile/多球并发弹射`、`mobile/挑战模式`、`mobile/波次推进与危险线`。
- L5：多球并发、棋盘波次和挑战模式架构。
- L7：多球波次、挑战模式与重球自动/手工验证。

## 7. 风险与待确认

无阻塞项。重球初始出现率采用 20% 的实现默认值，后续可直接在 `GameConfig` 调整；奖励块上线时应替换该临时初始抽取来源。
