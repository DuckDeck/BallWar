# 施工蓝图：T3 奖励块

> 上游输入：[T3-奖励块轻量设计方案](T3-奖励块轻量设计方案.md)  
> 状态：已归档

## 1. 任务边界

实现两种替代普通障碍的新行奖励格、即时球效果与生命周期库存；不改分数、奖励 UI 或存档。

## 2. 变更清单

### 新增

- `scripts/reward_block.gd`、`scenes/reward_block.tscn`：可碰撞、可绘制、命中即收集的奖励节点。
- `tests/t3_reward_block_test.gd`：概率、格位替代、两类效果和掉落球回归。

### 修改

- `scripts/wave_entry.gd`、`scripts/wave_generator.gd`：在每个新行格位抽取普通障碍或奖励类别。
- `scripts/board_controller.gd`、`scenes/main.tscn`：统一登记奖励/障碍棋盘节点并转发收集事件。
- `scripts/game_config.gd`：奖励类别概率与上限调参。
- `scripts/ball.gd`、`scripts/ball_manager.gd`、`scripts/game_controller.gd`：重球升级、活动批次掉落球和永久库存。
- L1/L2/L5/L7：回补已拍板规则与验收。

## 3. 关键实现点

- 奖励节点收集后先禁用碰撞，再通知 Controller；命中球不会获得反弹结果。
- 掉落球不是待发预览，必须进入当前 `BallLaunchSequence.active_balls`，以保证批次未完整回收前不结算。
- 普通球升级只修改该球的定义并同步碰撞半径/拖尾；重球复制新定义给掉落球。

## 5. 历史资产退出

不保留奖励与普通障碍叠加的兼容路径；一格只登记一个棋盘节点。

## 6. 红线自检

- [x] 不改变得分与危险线终止的职责归属。
- [x] 不引入 Autoload、存档或跨局状态。
- [x] 不让奖励块绕过 `BallManager` 批次所有权。

## 7. 验收映射

| 已拍板规则 | 落点 |
|---|---|
| +1 即时且永久 | `GameController`/`BallManager` 与奖励专项测试。 |
| 普通升级、重球复制 | `Ball`/`GameController` 与奖励专项测试。 |
| 立即消失、无分无反弹 | `RewardBlock.receive_hit`。 |
| 20%/10% 格位替代 | `WaveGenerator` 与生成测试。 |

## 8. 验证与回归

运行奖励专项、T2/T3 架构、挑战模式、障碍多样性及基础物理回归；手工确认奖励图标与参考图语义一致。

## 9. 风险与回滚

若奖励频率影响节奏，将两个概率调为 0 即可退回纯障碍行；无需移除棋盘/批次结构。

## 10. 可开工性结论

可以。奖励类型、命中、库存、生成概率和格位关系均已由用户确认。
