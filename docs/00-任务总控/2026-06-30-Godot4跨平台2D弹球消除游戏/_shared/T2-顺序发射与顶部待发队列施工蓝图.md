> 上游输入：用户于 2026-07-19 确认——首球发射后，后续球按同一瞄准角度依次发射；未发射球显示在顶部两条斜线构成的槽内。
> 状态：已归档

## 1. 任务边界

将现有“同帧创建并发射整批球”替换为“首球立即发射、剩余球按固定短间隔依次发射”。待发球仅为 Launcher 的视觉预览，不创建物理实体；已发射球仍彼此不碰撞，且全批次所有球回收后才结算回合。

不改变障碍命中、棋盘推进、回收路径、特殊球效果或暂停/结算 UI。

## 2. 变更清单

### 新增

- `scripts/ball_launch_sequence.gd`：持有一个批次的待发定义、已发射球和发射节拍。

### 修改

- `scripts/game_config.gd`、`resources/game_config.tres`：增加顺序发射间隔，默认关闭扇形偏转。
- `scripts/ball_manager.gd`：按队列首发与后续定时发射，并向 UI 发布待发队列。
- `scripts/ball_batch.gd`：每颗球沿同一基准瞄准方向发射。
- `scripts/game_controller.gd`、`scripts/main.gd`、`scripts/launcher.gd`：把下一批和当前待发队列同步到顶部槽位预览。
- `tests/t2_t3_architecture_test.gd`：验证首帧仅发一球、待发队列递减、所有球最终才结算。
- `docs/03-前端技术/godot/L5-多球并发架构.md`、`docs/04-测试/04-01-手工验证场景/多球与棋盘波次/runbook.md`：同步发射语义与手工验收。

## 3. 关键实现点

- `BallLaunchSequence` 是 Manager 的批次内部状态；它不进入场景树，待发球不会参与物理。
- 每次从队列取球都使用原始 `direction`，不使用扇形偏转；队列变化立即驱动 Launcher 重绘。
- `batch_finished` 的条件是“待发队列为空且已发射球注册表为空”，因此不会提前推进棋盘。
