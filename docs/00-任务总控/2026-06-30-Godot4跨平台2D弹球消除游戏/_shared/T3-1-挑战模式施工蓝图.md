## 1. 任务边界

实现启动时的经典/挑战模式选择、挑战模式独立 10 秒棋盘时钟、活动球期间的方块上移和临时每波次加一球。经典模式保留“球全部回收后推进棋盘”的现有行为。

不实现账号、分享、货币、商城、排行、设置、奖励块、模式存档、暂停菜单或最终结算界面；参考图仅作为深色首页和双主按钮的视觉方向。

## 2. 变更清单

### 新增

- `scripts/game_mode_definition.gd` 与两个模式 Resource：模式数据、波次触发策略和挑战间隔。
- `scripts/challenge_wave_clock.gd`：仅发出定时波次到期和倒计时变化信号。
- `scripts/mode_selection.gd`：菜单按钮的样式与模式选择信号。
- `docs/04-测试/04-02-自动化测试/挑战模式.md`、`tests/t3_challenge_mode_test.gd`：挑战模式自动验收说明和执行资产。

### 修改

- `scenes/main.tscn`：添加 `ChallengeWaveClock`、模式选择 Control、挑战倒计时 HUD；将 Controller 与时钟连线。
- `scripts/main.gd`：启动时仅显示模式菜单；收到选择后显示游戏区域并启动指定模式。
- `scripts/game_controller.gd`：将启动、批次完成和定时波次分流；定时波次在物理边界提交，失败时冻结活动球。
- `scripts/ball.gd`、`scripts/ball_manager.gd`：提供仅供 `GAME_OVER` 使用的活动球冻结入口。
- `scripts/board_controller.gd`：提取统一的上移/危险线/补行事务，支持批次和定时触发。
- `scripts/game_config.gd`、`resources/game_config.tres`：集中挑战波次间隔与临时加球配置。
- L1/L2/L5/L7 与任务总控：回填已实现入口、参数和验收引用。

### 删除

- 不删除现有经典模式、`ProgressionState` 或奖励块占位边界；旧安全回合自动加球只停止作为挑战模式依赖。

## 3. 关键实现点

- `GameController` 初始进入 `MODE_SELECTION`，不初始化棋盘或计时；模式选择后才创建首行并进入 `READY`。
- 挑战时钟到期只设置“待处理波次”；由 `GameController._physics_process()` 调用棋盘事务，确保方块格坐标、节点位置和碰撞体一次性提交。
- 棋盘事务固定为“上移旧方块 → 危险线 → 安全时按传入的下一批球数生成底行”。挑战批次完成只恢复 `READY`，不会调用该事务。
- 奖励块未实施时，挑战模式成功定时波次将下一批临时球数加一；经典模式继续兼容现有增长逻辑。
- 挑战定时波次先提交旧方块的逻辑行号并判定危险线；安全时以 `0.4` 秒 Tween 同步上移旧方块与活动球，动画结束后再生成底行并恢复球物理。经典模式保持原有即时推进。

## 5. 历史资产退出

`BoardController.resolve_completed_batch()` 的内部上移/补行逻辑抽为统一事务；保留其公开入口供经典模式使用。`ProgressionState.safe_turns` 不再参与挑战球数计算，奖励块阶段再统一移除自动加球。

## 6. 红线自检

- [x] 不引入 Autoload；模式、时钟和棋盘仍属于当前主场景。
- [x] `BallManager` 不负责棋盘时钟，`BoardController` 不读取活动球。
- [x] 所有新增 GDScript 变量、参数、返回值和 signal 显式标注类型。
- [x] 不触及支付、身份、数据删除等死亡线区域。

## 7. 验收映射

| 已拍板决策 | 施工落点 |
|---|---|
| 开局立即生成首行 | `GameController.start_game()` → `BoardController.initialize_board()` |
| 10 秒独立推进 | `ChallengeWaveClock` → `GameController._physics_process()` |
| 球飞行时仍推进 | 统一棋盘事务不检查 `BallManager` 活动球数 |
| 批次完成仅恢复发射 | 挑战分支的 `_on_batch_finished()` |
| 暂时每波次加一球 | 挑战模式临时球数策略 |
| 启动前选择模式 | `ModeSelection` 与 `Main` |

## 9. 风险与回滚

若挑战路径异常，选择经典模式仍走原批次结算入口。所有挑战配置和入口都局限在新模式定义/时钟分支，可关闭挑战按钮或移除挑战 Resource 回退，不影响球物理与普通棋盘格真值。

## 10. 可开工性结论

可以。用户已明确模式规则、首行时机、定时推进、活动球并发与临时加球策略；未确认的账号/商城等参考图内容已列为非范围。
