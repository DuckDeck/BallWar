# L5 棋盘波次与结算架构

> 本文落实底部随机单行、整体上移、危险线失败和难度增长规则。  
> 版本归属：V0.3 | 生命周期状态：已审核

## 设计结论

方块推进采用逻辑棋盘，而不是直接将 `Obstacle.global_position.y` 加减像素。`BoardState` 保存格坐标和占用关系；`BoardLayout` 将格坐标映射到场地坐标；`BoardController` 以统一事务提交一次波次推进。普通模式在没有活动球时由批次结算触发；挑战模式可在球活动时由独立时钟触发，详细边界见 [L5-挑战模式与棋盘时钟架构.md](L5-挑战模式与棋盘时钟架构.md)。

随机生成由 `WaveGenerator` 独立负责，并持有本局 `RandomNumberGenerator`。它为每个 `WaveEntry` 一并生成列、生命值、形状枚举和中心旋转角；生命值按下一回合球数的 `1×～2×` 打散，优先给同一行不同数字。`GameController` 不选择随机列、不计算数字，也不逐个移动方块；它只在 `BallManager.batch_finished` 后驱动结算顺序。

## 当前实现锚点

- `BoardState` 是 `(column,row)` 占用关系的唯一真值；逻辑行号增加表示向顶部推进，`BoardLayout` 再将其映射为屏幕 Y 坐标。
- `BoardController` 初始化时调用 `WaveGenerator` 生成至少一个底行方块；每个安全批次按“上移 → 危险线 → 成长 → 生成底行”结算一次。
- 现有 `ProgressionState` 的安全回合自动加球仅是待替换实现；长期由奖励块维护球库存。挑战模式奖励块上线前使用独立的定时波次临时加球策略，不能再把球数成长绑定为安全回合数。随机种子同样由 `GameConfig.wave_seed` 固定。
- `Obstacle` 根据 `WaveEntry.shape_type` 在运行时创建匹配的 `CircleShape2D` 或 `ConvexPolygonShape2D`；节点旋转负责形状与碰撞同步旋转，绘制时反向抵消数字旋转以保持可读。`Obstacle.receive_hit(HitContext)` 是统一受击入口。销毁时 `BoardController` 同步释放棋盘占用并向 `GameController` 发出唯一得分事件。

## 运行时对象

```text
普通模式：GameController
  └─ BallManager.batch_finished
       → BoardController.resolve_completed_batch
            → BoardState.advance_rows(1)
            → DangerLine 判定
            → ProgressionState.advance（下一回合球数/数字）
            → WaveGenerator.spawn_bottom_row（仅安全时）

BoardController
├── BoardState（逻辑格、占用、回合）
├── BoardLayout（列/行到场地坐标）
├── WaveGenerator（种子随机的新底行）
└── ObstacleLayer（方块节点视图与碰撞体）
```

| 对象 | 负责 | 禁止负责 |
|---|---|---|
| `BoardState` | `column,row`、占用、存活方块 ID。 | Node 引用、绘制、Tween、难度计算。 |
| `BoardLayout` | 横向列数、格宽/格高、底行基准、危险线和场地坐标换算。 | 随机生成和分数。 |
| `BoardController` | 将逻辑推进同步给方块节点、危险线判断、一次性波次结算。 | 球的物理或多球注册。 |
| `WaveGenerator` | 在底行挑选不重复列、随机生成形状和中心旋转，并按下一回合球数生成 `1×～2×` 数字。 | 推进旧方块或结束回合。 |
| `ProgressionState` | 临时承载现有普通模式球数成长；奖励块上线后迁移为模式无关的球库存/过渡策略。 | 实例化场景、随机抽样或形状选择。 |
| `Obstacle` | 呈现一个格内方块、受击、销毁并向 BoardController 报告。 | 决定网格位置或自行上移。 |

## 棋盘与危险线

`BoardLayout` 的横向列数由可用场地宽度和单格宽度得出。每个格子一行高，底部新行固定为 `row = 0`；整体上移时所有存活方块的 `row += 1`。同一行的新方块列不重复，生成数不超过列总数，故天然满足“最多一排”。

危险线判定采用方块上边缘：将逻辑格转换到世界坐标后，方块上边缘 `<= danger_line_y` 即失败。不要用方块中心点，也不要以渲染动画的位置判断。触发失败后 BoardController 不调用 WaveGenerator。

## 普通模式的回合事务边界

在普通模式中，`BallManager.batch_finished` 是触发波次推进的唯一事件；该信号已保证所有活动球均实际回到顶部发射点。BoardController 对同一个 `batch_id` 只接受一次结算：

1. 提交所有方块的逻辑上移并同步节点；
2. 判定危险线；
3. 失败则发出 `game_over(final_score, reached_turn)`；安全则推进 `ProgressionState` 并生成底行；
4. 仅安全路径通知 Controller 恢复 `READY`。

普通模式的方块移动动画只能在这个事务已经提交后播放。因为事务开始前活动球集合已为空，动画不会与球碰撞并发；挑战模式的活动球并发推进要求见 [L5-挑战模式与棋盘时钟架构.md](L5-挑战模式与棋盘时钟架构.md)。

## 难度与随机性

新方块数字由下一批可控球数的 `1×～2×` 范围生成。长期球数由奖励块改变；过渡期间的临时加球必须由模式策略给出，不能把“安全回合数”当作跨模式真值。具体上限和倍率放在 Resource 配置而不是 Controller 分支。`WaveGenerator` 使用显式本局种子；测试可固定种子断言“某回合的列布局、方块数、形状、旋转和数字”。

基础增长的正式来源是奖励物，而不是安全回合或自然经过时间。奖励物将修改下一批的球库存；挑战模式的“每个定时波次临时加一球”只在奖励物缺席期间有效，且不得绕过棋盘推进顺序。

## 与多球和特殊内容的关系

- 普通模式在所有球回收后才推进棋盘；挑战模式的定时推进不等待球回收，单颗球提前回收也不影响定时器。
- 每个方块无论形状如何都占用一个逻辑格；视觉/碰撞形状在格内变化，不改变一格推进语义。
- 特殊球可通过统一命中协议影响多个方块，但方块销毁必须同步从 BoardState 移除占用，防止下一次上移出现幽灵格。
- 游戏结束由 Controller 状态 `GAME_OVER` 和 HUD 结算承接；BoardController 不直接绘制结算 UI。

## 测试边界

- 随机底行：不重复列、至少一块、至多一排、六种形状与中心旋转、固定种子可复现。
- 普通模式回合推进：所有球回收前不移动；全部回收后恰好上移一格并只生成一行。
- 危险线：方块上边缘触线/越线均触发失败；失败时不生成新行、不增加球数。
- 难度：连续安全回合的球数、数字最小值/最大值均不下降，并受上限约束。
- 同一批次多球同帧回收时，普通模式棋盘只推进一次；挑战模式另行验证定时波次不因批次完成而重复触发。
