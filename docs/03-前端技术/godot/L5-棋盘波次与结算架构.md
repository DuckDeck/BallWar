# L5 棋盘波次与结算架构

> 本文落实底部随机单行、整体上移、危险线失败和难度增长规则。  
> 版本归属：V0.3 | 生命周期状态：已审核

## 设计结论

方块推进采用逻辑棋盘，而不是直接将 `Obstacle.global_position.y` 加减像素。`BoardState` 保存格坐标和占用关系；`BoardLayout` 将格坐标映射到场地坐标；`BoardController` 在没有活动球时提交一次波次推进。这样能稳定处理“一格上移”“顶部触线”“一排容量”和未来的不同方块形状。

随机生成由 `WaveGenerator` 独立负责，并持有本局 `RandomNumberGenerator`。`GameController` 不选择随机列、不计算数字，也不逐个移动方块；它只在 `BallManager.batch_finished` 后驱动结算顺序。

## 当前实现锚点

- `BoardState` 是 `(column,row)` 占用关系的唯一真值；逻辑行号增加表示向顶部推进，`BoardLayout` 再将其映射为屏幕 Y 坐标。
- `BoardController` 初始化时调用 `WaveGenerator` 生成至少一个底行方块；每个安全批次按“上移 → 危险线 → 成长 → 生成底行”结算一次。
- `ProgressionState` 只记录安全回合数，并通过 `GameConfig` 的上限参数提供下一批球数和方块血量成长；随机种子同样由 `GameConfig.wave_seed` 固定。
- `Obstacle.receive_hit(HitContext)` 是统一受击入口。销毁时 `BoardController` 同步释放棋盘占用并向 `GameController` 发出唯一得分事件。

## 运行时对象

```text
GameController
  └─ BallManager.batch_finished
       → BoardController.resolve_turn
            → BoardState.advance_rows(1)
            → DangerLine 判定
            → WaveGenerator.spawn_bottom_row（仅安全时）
            → ProgressionState.advance（下一回合球数/数字）

BoardController
├── BoardState（逻辑格、占用、回合）
├── BoardLayout（列/行到场地坐标）
├── WaveGenerator（种子随机的新底行）
└── ObstacleLayer（方块节点视图与碰撞体）
```

| 对象 | 负责 | 禁止负责 |
|---|---|---|
| `BoardState` | `column,row`、占用、存活方块 ID、当前安全回合。 | Node 引用、绘制、Tween。 |
| `BoardLayout` | 横向列数、格宽/格高、底行基准、危险线和场地坐标换算。 | 随机生成和分数。 |
| `BoardController` | 将逻辑推进同步给方块节点、危险线判断、一次性波次结算。 | 球的物理或多球注册。 |
| `WaveGenerator` | 依据难度在底行挑选不重复列、生成一个或多个方块和数字。 | 推进旧方块或结束回合。 |
| `ProgressionState` | 根据安全回合给出下一批球数、方块数量范围和数字范围。 | 实例化场景或随机抽样。 |
| `Obstacle` | 呈现一个格内方块、受击、销毁并向 BoardController 报告。 | 决定网格位置或自行上移。 |

## 棋盘与危险线

`BoardLayout` 的横向列数由可用场地宽度和单格宽度得出。每个格子一行高，底部新行固定为 `row = 0`；整体上移时所有存活方块的 `row += 1`。同一行的新方块列不重复，生成数不超过列总数，故天然满足“最多一排”。

危险线判定采用方块上边缘：将逻辑格转换到世界坐标后，方块上边缘 `<= danger_line_y` 即失败。不要用方块中心点，也不要以渲染动画的位置判断。触发失败后 BoardController 不调用 WaveGenerator。

## 回合事务边界

`BallManager.batch_finished` 是唯一可触发波次推进的事件。BoardController 对同一个 `batch_id` 只接受一次结算：

1. 提交所有方块的逻辑上移并同步节点；
2. 判定危险线；
3. 失败则发出 `game_over(final_score, reached_turn)`；安全则推进 `ProgressionState` 并生成底行；
4. 仅安全路径通知 Controller 恢复 `READY`。

方块移动动画只能在这个事务已经提交后播放。因为事务开始前活动球集合已为空，动画不会与球碰撞并发。

## 难度与随机性

`ProgressionState` 以安全回合为输入，提供单调不减的球数和新方块数字范围；具体公式放在 Resource 配置而不是 Controller 分支。`WaveGenerator` 使用显式本局种子；测试可固定种子断言“某回合的列布局、方块数和数字”。

基础增长来自安全回合，而不是奖励物。未来奖励物可以修改下一回合的 `ProgressionState` 输入或提供临时加球，但不得绕过批次结算和棋盘推进顺序。

## 与多球和特殊内容的关系

- 所有球回收后才推进棋盘；单颗球提前回收不触发上移。
- 每个方块无论形状如何都占用一个逻辑格；视觉/碰撞形状在格内变化，不改变一格推进语义。
- 特殊球可通过统一命中协议影响多个方块，但方块销毁必须同步从 BoardState 移除占用，防止下一次上移出现幽灵格。
- 游戏结束由 Controller 状态 `GAME_OVER` 和 HUD 结算承接；BoardController 不直接绘制结算 UI。

## 测试边界

- 随机底行：不重复列、至少一块、至多一排、固定种子可复现。
- 回合推进：所有球回收前不移动；全部回收后恰好上移一格并只生成一行。
- 危险线：方块上边缘触线/越线均触发失败；失败时不生成新行、不增加球数。
- 难度：连续安全回合的球数、数字最小值/最大值均不下降，并受上限约束。
- 同一批次多球同帧回收时，棋盘只推进一次。
