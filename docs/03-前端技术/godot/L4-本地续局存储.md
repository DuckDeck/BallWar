# L4 本地续局存储

> 版本归属：V0.4 | 生命周期状态：已审核

## 文件

续局数据写入 `user://game_sessions.cfg`，与 `user://score_records.cfg` 分离。每个模式各占一个 section：`classic`、`challenge`。

## Section 字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `schema_version` | `int` | 当前固定为 `1`；不兼容版本直接丢弃该 section。 |
| `snapshot` | `Dictionary` | 模式、分数、回合、正计时、库存球定义、棋盘节点与模式时钟。 |

`snapshot.board` 为节点数组，每项保存 `kind`、逻辑格 `column/row`、健康值/奖励类型、形状和旋转；不保存运行时 Node 实例或 Tween。球定义保存普通/重球类型、半径、重力、伤害概率和颜色 HTML 值。

## 数据不变量

- 只接受 `classic` 或 `challenge` section，且 snapshot 模式必须与 section 一致。
- 经典模式 snapshot 只由 `READY` 检查点产出；挑战模式 snapshot 将所有待发、排队和飞行球定义收敛为顶部库存。
- `ConfigFile` 读写错误、格式错误或版本不符时，只删除出错 section。
