# L5 HUD 与续局架构

> 版本归属：V0.4 | 生命周期状态：已审核

`HUD` 是独立 `Control` 场景，仅公开显示方法与 `pause_requested` 信号；`Main` 继续作为 UI 到 `GameController` 的编排边界。布局计算只依赖 HUD 自身尺寸，因此画布缩放或桌面窗口尺寸变化不会改变各 HUD 区块的相对锚点。

`GameController` 持有续局快照的业务语义。经典模式每次达到 `READY` 后更新内存检查点；挑战模式在请求保存时即时组合棋盘和所有球定义。恢复始终创建新的 `READY` 对局，不恢复活动球/发射序列；挑战时钟从保存的剩余时长重启，不计算离线时间。

`GameSessionStore` 是 `ConfigFile` 的唯一访问边界。`Main` 在稳定检查点、挑战模式有首次发射或定时波次推进之后的秒级时钟更新、后台/失焦/退出通知和用户点击保存退出时请求持久化。刚进入挑战但未发生玩法进度时，快照为空，保存退出仍返回模式选择且不会触发续局确认。模式主按钮先经 `Main` 查询同模式快照：有快照则将其原始整数分数交给 `ProgressDialog` 展示；“继续”恢复快照，“不继续”清除后新开局。存档恢复和清除旧档始终只通过该边界操作。

约束：棋盘快照由 `BoardController` 负责导出/重建，球定义序列化由 `GameController` 负责；Store 不持有场景节点或玩法对象引用。
