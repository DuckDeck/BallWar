# L5 游戏音频架构

> 版本归属：V0.4 | 生命周期状态：已审核

`GameAudio` 以 `AudioManager` 名称注册为 Autoload。它仅持有一个循环 `AudioStreamPlayer` 和预创建的音效播放器池，不保存任何局内节点引用。

`Main` 负责模式 BGM 与暂停/结束/退出的生命周期；`BallManager`、`Obstacle`、`RewardBlock` 仅在自己的领域事件发生时请求全局音效。命中和回收的短冷却由 `GameAudio` 统一处理，避免多球实现向玩法层泄漏音频节流逻辑。

| 参数 | 默认值 | 用途 |
|---|---:|---|
| `music_volume_db` | `-18 dB` | BGM 音量 |
| `sfx_volume_db` | `-8 dB` | 音效音量 |
| `repeated_sfx_cooldown_seconds` | `0.05 s` | 命中/回收限流 |
| `sfx_pool_size` | `8` | 可并发音效数量 |

音频统一置于 `assets/audio/`，使用 Godot 的 WAV 导入产物，Android 导出时随项目资源打包。BGM 循环由管理器在运行时显式设置，避免依赖单个导入配置。
