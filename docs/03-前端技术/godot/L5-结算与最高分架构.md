# L5 结算与最高分架构

> 版本归属：V0.4 | 生命周期状态：已审核

## 设计结论

`GameController` 仍是终局事实的唯一来源：危险线触发后进入 `GAME_OVER`、停止挑战时钟、冻结活动球，并通过 `game_over(final_score, reached_turn)` 发布最终成绩。`Main` 接收该事件，按当前模式写入成绩记录、展示 `GameOverPanel`，然后设置 `SceneTree.paused` 冻结场景。

`ScoreRecordStore` 是唯一的本地成绩读写边界，使用 `user://score_records.cfg`。它将经典和挑战模式存放在独立 section；每个 section 包含 `historical_best`、`daily_best`、`daily_date`。读取或写入时若 `daily_date` 与设备本地日期不同，今日分数按零开始比较，历史分数不受影响。

`GameOverPanel` 仅负责显示和发射 `restart_requested`、`menu_requested`，自身使用 `PROCESS_MODE_ALWAYS`，不直接改变游戏状态。`Main` 在执行菜单动作时先解除树暂停：重开调用 `GameController.restart_game()`，该方法接受 `PAUSED` 与 `GAME_OVER`；返回调用 `GameController.return_to_mode_selection()`，统一清理球、棋盘、时钟、库存队列和 HUD 数值后显示既有 `ModeSelection`。

## 约束

- 成绩记录只在终局时写入；重开、返回和暂停不得覆盖成绩。
- 分享按钮保留 disabled 视觉状态，不得连接业务回调。
- `ScoreRecordStore` 不存储活动对局，因此不会与后续续局存档的职责重叠。
