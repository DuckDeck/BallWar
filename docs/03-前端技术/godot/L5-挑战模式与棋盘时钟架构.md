# L5 挑战模式与棋盘时钟架构

> 本文锁定挑战模式的独立棋盘时钟、球批次边界与普通模式隔离策略。  
> 版本归属：V0.4 | 生命周期状态：已审核

## 设计结论

挑战模式不能通过在现有 `batch_finished` 回调上增加条件分支实现。球批次与棋盘波次是两条独立时间线：`BallManager` 只决定何时重新允许发射；`ChallengeWaveClock` 只决定何时请求棋盘推进。两者都通过 `GameController` 编排，并只调用 `BoardController` 的统一波次事务。

`GameModeDefinition` 已以 `Resource` 落地为 `classic_mode.tres` 和 `challenge_mode.tres`，而不是在多个 Controller 中传播 `challenge_mode: bool`。模式定义提供推进策略、定时波次间隔和临时球数策略；选择页只传递模式枚举，`GameController` 持有实际 Resource。

## 核心链路

```text
挑战模式开局
  → BoardController 初始化第一行
  → ChallengeWaveClock 开始 10 秒循环

球批次
  → BallManager.batch_finished
  → GameController 只恢复 READY 与待发预览

定时波次（无论是否有活动球）
  → ChallengeWaveClock.wave_due
  → GameController 提交 BoardController.resolve_timed_wave()
  → 逻辑格上移并判危险线 → 在底部外一格创建新底行
  → 新旧方块与活动球同步滚屏 0.4s → 重启 10 秒周期
```

## 职责边界

| 对象 | 负责 | 禁止负责 |
|---|---|---|
| `GameMode` / `GameModeDefinition` | 声明普通或挑战模式、推进策略和可调参数。 | 保存活动球、障碍节点或 UI 引用。 |
| `ChallengeWaveClock` | 仅累计/发出定时波次到期事件；暂停和 `GAME_OVER` 时停止。 | 直接移动方块、生成球或修改分数。 |
| `GameController` | 根据模式路由批次完成与定时波次，并在 `GAME_OVER` 时统一停止时钟和输入。 | 自行计算格坐标、随机列或障碍数字。 |
| `BoardController` | 以单一原子事务执行“上移 → 危险线 → 生成底行”，接受 `BATCH` 或 `TIMED` 触发原因。 | 判断球是否已全部回收，或管理计时器。 |
| `BallManager` | 顺序发射队列、活动球集合与单球回收信号；挑战模式将回收球交回发射槽，经典模式仍在最后一球回收后完成批次。 | 推进挑战倒计时或棋盘。 |
| `BallInventory`（奖励块阶段） | 保存玩家已获得的球数并构造下一批球定义。 | 决定波次推进或直接生成奖励块。 |

当前场景额外包含 `ModeSelection` 和 `ChallengeWaveClock`。`Main` 在 `MODE_SELECTION` 阶段隐藏场地与局内 HUD；接收菜单选择后调用 `GameController.start_game_by_mode_id()`。`ChallengeWaveClock` 只发出到期信号，`GameController` 在下一物理帧提交棋盘事务，并通过 HUD 的 `Wave: NN s` 显示倒计时。

## 并发与事务约束

- `ChallengeWaveClock` 到期时只发出 `wave_due`；棋盘变更必须由 `BoardController` 在物理安全边界一次性提交，避免同一帧一边更新碰撞形状一边半完成格坐标。
- 允许棋盘事务与活动球并存，但一次事务内部必须先提交全部逻辑格并完成危险线判定；安全时在底部外一格创建 `row=0` 新方块，再将新旧障碍节点、碰撞体和所有活动球按同一 `QUAD / EASE_OUT` 曲线滚动至目标位置，动画时长由模式资源配置（当前 `0.4s`）。球物理在滚屏内保持运行，并在每个物理帧叠加同曲线的场地位移；不得逐个方块跨多帧提交。
- 定时波次生成顺序固定为“逻辑上移旧方块，失败则停止，安全时创建场外 `row=0` 新方块并整体滚屏”。同一到期事件只允许执行一次，滚屏中拒绝再次提交。
- `GAME_OVER` 高于未完成球批次和下一次定时波次：立即停止时钟、拒绝新发射，并由场景状态统一冻结后续活动。

## 球数来源迁移

长期方案是奖励块更新 `BallInventory`，批次创建仅读取库存；普通/挑战模式都不得按回合或时间直接增加球数。奖励块尚未实施时，挑战模式可在每个成功 `TIMED` 波次后通过可替换的过渡策略发放一颗球。该策略必须封装在模式/进程对象中，不能继续散落在 `GameController` 或 `ProgressionState.safe_turns`。

## 验收锚点

- 开局第一行出现，首次定时波次恰在 10 秒到期时执行。
- 球仍活动时，定时波次仍只推进一次，且球不被暂停、删除或重复结算。
- 最后一颗球回收只恢复发射权限，不移动棋盘、不重置挑战倒计时。
- 危险线触发后，无论是否仍有活动球，都不再出现新行或下一次倒计时。
