# 挑战模式需求

> 所属功能域：挑战模式、定时波次与失败结算  
> 版本归属：V0.4 | 生命周期状态：已审核

## 目标

挑战模式将“发射槽中的待发球”和“棋盘推进”分离：任一球回到顶部发射点都会进入待发槽并立即恢复其发射资格，不推动棋盘；棋盘从开局起按固定时间间隔自行推进，形成在球仍飞行时也持续上升的生存压力。

## 模式入口

- 游戏启动时先显示模式选择页，不创建棋盘、不启动局内计时、也不接受发射输入。
- 用户可选择“经典模式”或“挑战模式”；经典模式沿用完整球批次结算后推进棋盘的规则。
- 菜单只负责选择本局模式；账号、分享、货币、商城、排行榜和设置入口不属于本阶段。

## 核心规则

- **L1-CH-01**：挑战模式开局立即生成一行底部随机障碍；第一轮定时推进从开局开始计时，不额外等待首行生成。
- **L1-CH-02**：挑战模式每隔 `10` 秒执行一次定时波次。定时波次不依赖发射次数、瞄准状态或球是否仍在场内。
- **L1-CH-03**：每次定时波次先提交“已有方块整体上移一格”的逻辑格并判定危险线；安全时先在场地底部外一格生成新底行，再让新旧方块与所有活动球在 `0.4s` 内同步向上平移一格。新生成的底行在动画结束时恰好抵达 `row=0`。
- **L1-CH-04**：球仍处于 `BALLS_ACTIVE`、托底回收或顶部回程时，定时波次仍可执行；滚屏期间活动球随场地同步平移，同时持续执行自身重力、反弹、回收与拖尾逻辑，不得暂停物理推进。
- **L1-CH-05**：任一方块上边缘触及或越过顶部危险虚线时立即 `GAME_OVER`；停止后续定时波次、禁用发射、冻结分数。
- **L1-CH-06**：所有球实际回到顶部发射点后，只恢复 `READY` 和下一批发射资格；不得因此生成方块、移动棋盘或重置定时波次倒计时。

## 球数与奖励规则

- **L1-CH-07**：两种模式的新增球仅允许来自奖励块；球数不以发射回合数、成功波次或经过时间作为成长来源。`+1` 奖励会立即生成自然下落的普通球并永久加入本局后续库存；重球命中放大奖励时立即自然下落一颗重球并永久加入库存。普通球命中放大奖励只原地升级为重球，不增加库存。
- **L1-CH-09**：挑战模式不要求场内全部球回到发射点才允许再次发射；只要发射槽中存在待发球，即可接受新的瞄准并启动新的顺序发射队列，已在场球继续独立运动。

## 与普通模式的边界

| 规则 | 普通模式 | 挑战模式 |
|---|---|---|
| 下一次发射 | 全部球实际回到发射点后恢复 | 任一待发球出现在发射槽后即可恢复 |
| 棋盘推进触发 | 一次完整球批次结算 | 固定 10 秒定时波次 |
| 球飞行时方块上移 | 不发生 | 允许发生 |
| 开局底行 | 立即生成 | 立即生成 |
| 正式新增球来源 | `+1` 奖励与重球命中放大奖励 | `+1` 奖励与重球命中放大奖励 |

## 非范围

- 不定义奖励块的音画反馈细节、存档或更多奖励类型。
- 不定义挑战模式选择页、结算 UI、排行榜、存档或难度档位。
- 不将“每 10 秒临时加一球”视为长期数值设计。
