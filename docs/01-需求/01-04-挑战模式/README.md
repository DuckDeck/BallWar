# 挑战模式需求

> 所属功能域：挑战模式、定时波次与失败结算  
> 版本归属：V0.4 | 生命周期状态：已审核

## 目标

挑战模式将“下一次发射资格”和“棋盘推进”分离：球全部回到顶部发射点只恢复下一次发射，不推动棋盘；棋盘从开局起按固定时间间隔自行推进，形成在球仍飞行时也持续上升的生存压力。

## 模式入口

- 游戏启动时先显示模式选择页，不创建棋盘、不启动局内计时、也不接受发射输入。
- 用户可选择“经典模式”或“挑战模式”；经典模式沿用完整球批次结算后推进棋盘的规则。
- 菜单只负责选择本局模式；账号、分享、货币、商城、排行榜和设置入口不属于本阶段。

## 核心规则

- **L1-CH-01**：挑战模式开局立即生成一行底部随机障碍；第一轮定时推进从开局开始计时，不额外等待首行生成。
- **L1-CH-02**：挑战模式每隔 `10` 秒执行一次定时波次。定时波次不依赖发射次数、瞄准状态或球是否仍在场内。
- **L1-CH-03**：每次定时波次按“已有方块整体上移一格 → 危险线判定 → 安全时在底部生成一行”的顺序执行；新生成的底行不得在同一次波次中再次上移。
- **L1-CH-04**：球仍处于 `BALLS_ACTIVE`、托底回收或顶部回程时，定时波次仍可执行；方块上移与球的物理运动可以并发。
- **L1-CH-05**：任一方块上边缘触及或越过顶部危险虚线时立即 `GAME_OVER`；停止后续定时波次、禁用发射、冻结分数。
- **L1-CH-06**：所有球实际回到顶部发射点后，只恢复 `READY` 和下一批发射资格；不得因此生成方块、移动棋盘或重置定时波次倒计时。

## 球数与奖励规则

- **L1-CH-07（正式规则）**：两种模式的新增球应来自障碍行中按概率生成的奖励块；球数不以发射回合数或经过时间作为正式成长来源。奖励块的外观、概率、拾取判定和持久化效果留待后续功能定义。
- **L1-CH-08（过渡规则）**：奖励块尚未实现期间，挑战模式每次成功完成一个 `10` 秒定时波次后临时增加 `1` 颗可控球，受球数上限限制；新底行数字以增加后的下一批球数计算。该过渡规则将在奖励块上线时移除。
- 普通模式现有的“安全回合自动加球”同样属于待替换的旧实现；后续奖励块功能完成时，两种模式统一切换到 L1-CH-07，不再新增另一套自动加球规则。

## 与普通模式的边界

| 规则 | 普通模式 | 挑战模式 |
|---|---|---|
| 下一次发射 | 全部球实际回到发射点后恢复 | 全部球实际回到发射点后恢复 |
| 棋盘推进触发 | 一次完整球批次结算 | 固定 10 秒定时波次 |
| 球飞行时方块上移 | 不发生 | 允许发生 |
| 开局底行 | 立即生成 | 立即生成 |
| 正式新增球来源 | 奖励块（待实现） | 奖励块（待实现） |

## 非范围

- 不定义奖励块的具体概率、形状、碰撞效果、音画反馈或多种奖励类型。
- 不定义挑战模式选择页、结算 UI、排行榜、存档或难度档位。
- 不将“每 10 秒临时加一球”视为长期数值设计。
