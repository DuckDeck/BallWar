# 波次推进与危险线需求

> 所属功能域：方块波次、难度增长与失败结算  
> 版本归属：V0.3 | 生命周期状态：已审核

## 目标

方块以“底部随机一行 → 每回合整体上移一格 → 触及顶部虚线失败”的节奏推进。随着安全完成的回合增加，下一回合可操控球数量增多，底部新生成方块的数字也增大。

## 方块棋盘规则

- 场内方块采用逻辑网格。一个方块占用一个格子，单行最多占满横向可用格数，每列最多一个新方块。
- 游戏开始时生成一行底部随机方块；每个安全回合后再生成一行新的底部随机方块。
- 每行随机生成一个或多个方块，数量不得超过横向格数，且新行至少包含一个方块。
- 方块的位置以 `(column, row)` 逻辑坐标为真值；视觉位置和碰撞体由网格转换得到，不以自由像素坐标作为推进依据。
- 方块被销毁后，其格子为空；空格不会在同一回合被填补。

## 回合推进顺序

1. 本回合批次内所有球都从底部回收到顶部发球点。
2. 系统进入 `RESOLVING`，将场内所有存活方块整体上移一个格子。
3. 若任一方块的上边缘触及或越过顶部虚线危险线，立即进入 `GAME_OVER`，停止生成下一行和下一批球，并开始最终分数结算。
4. 若未触发危险线，回合数增加；按新的回合难度生成一行底部随机方块，并更新下一回合可发射球数。
5. 系统恢复 `READY`，允许玩家发射下一批球。

## 难度增长规则

- **L1-WB-01**：安全完成的回合数是球数量和新方块数字增长的唯一基础；增长参数可配置并设上限。
- **L1-WB-02**：下一回合可操控球数不得低于上一回合；新生成方块数字的最小值和最大值不得随安全回合下降。
- **L1-WB-03**：同一回合的方块随机结果由本局随机源生成；测试或调试必须能够固定随机种子复现该行布局与数字。
- **L1-WB-04**：每回合只执行一次“整体上移 + 危险线检查 + 底部新行生成”；无论多球在同一帧回收多少颗，都不得重复推进。

## 失败与结算

- **L1-WB-05**：危险线是顶部现有虚线。判断以方块上边缘接触或越过该线为准，不以方块中心点为准。
- **L1-WB-06**：触发 `GAME_OVER` 后，发射器不可继续发射，当前累计分数冻结并作为最终结算分数交给 HUD/结算界面。
- **L1-WB-07**：危险线失败优先于新行生成和球数增长；失败回合不再创建底部方块。

## 非范围

- 不定义具体方块形状、爆炸/穿透等特殊球规则、奖励物规则和最终结算 UI 版式。
- 不固定默认球数、数字公式、每行生成数量或横向格数；这些是配置与手感调参，不改变本需求的推进顺序。
