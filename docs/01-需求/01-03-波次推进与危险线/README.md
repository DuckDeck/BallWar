# 波次推进与危险线需求

> 所属功能域：方块波次、难度增长与失败结算  
> 版本归属：V0.3 | 生命周期状态：已审核

## 目标

普通模式的方块以“底部随机一行 → 完整球批次结算后整体上移一格 → 触及顶部虚线失败”的节奏推进。挑战模式的定时推进规则见[挑战模式需求](../01-04-挑战模式/README.md)。本局球库存从开局数量开始，只能通过奖励块永久增加。

## 方块棋盘规则

- 场内对象采用逻辑网格。一个普通障碍或奖励块占用一个格子，单行最多占满横向可用格数，每列最多一个新对象。
- 游戏开始时生成一行底部随机方块；每个安全回合后再生成一行新的底部随机方块。
- 每行随机生成至少 4 个占格对象（普通障碍或奖励块），数量不得超过横向格数。
- 方块的位置以 `(column, row)` 逻辑坐标为真值；视觉位置和碰撞体由网格转换得到，不以自由像素坐标作为推进依据。
- 方块被销毁后，其格子为空；空格不会在同一回合被填补。
- 前两行固定只生成普通障碍；从第 3 行开始，新行每个已选格位才按配置抽取 `+1` 奖励、放大奖励或普通障碍；两类概率分别由配置控制，奖励块与普通障碍互斥，不能在同一格叠放。奖励格不生成生命数字。

## 回合推进顺序

1. 本回合批次内所有球都从底部回收到顶部发球点。
2. 系统进入 `RESOLVING`，将场内所有存活方块整体上移一个格子。
3. 若任一方块的上边缘触及或越过顶部虚线危险线，立即进入 `GAME_OVER`，停止生成下一行和下一批球，并开始最终分数结算。
4. 若未触发危险线，生成一行底部随机方块；按固定当前球数计算新行数字，不新增球。
5. 系统恢复 `READY`，允许玩家发射下一批球。

## 难度增长规则

- **L1-WB-01**：普通模式中，一次安全完成的完整球批次只触发一次棋盘推进；它不是新增球数的正式来源。
- **L1-WB-02**：新生成普通障碍的数字由下一批可控球数计算；安全回合和自然经过时间不得新增球，只有奖励块可更新本局库存。
- **L1-WB-03**：正式对局每次开局必须使用新的波次随机种子，不得重复固定奖励格顺序；测试或调试可配置非零随机种子，以复现该局布局与数字。
- **L1-WB-04**：每回合只执行一次“整体上移 + 危险线检查 + 底部新行生成”；无论多球在同一帧回收多少颗，都不得重复推进。
- **L1-WB-08**：每个新障碍从六边形、五边形、正方形、等边三角形、圆形、菱形中随机选择一种，并围绕自身中心随机旋转；无论外观如何均只占用一个逻辑格。
- **L1-WB-09**：每个新障碍数字从下一回合可控球数的 `1×～2×`（含端点）中随机抽取；同一行优先使用不同数字，只有数值范围不足覆盖整行时才循环复用。
- **L1-WB-10**：第 3 行之前不得生成奖励格；从第 3 行开始，奖励格以配置的 `+1` 概率与放大概率替代普通障碍格。两项配置均可独立调节，每个格位最终只能生成一种内容。

## 失败与结算

- **L1-WB-05**：危险线是顶部现有虚线。判断以方块上边缘接触或越过该线为准，不以方块中心点为准。
- **L1-WB-06**：触发 `GAME_OVER` 后，发射器不可继续发射，当前累计分数冻结并作为最终结算分数交给 HUD/结算界面。
- **L1-WB-07**：危险线失败优先于新行生成和任何奖励结算；失败回合不再创建底部方块或新增球。

## 非范围

- 不定义爆炸/穿透等特殊球规则和最终结算 UI 版式。
- 不固定默认球数或数字公式；这些是配置与手感调参，不改变本需求的推进顺序。每行最少 4 个占格对象和横向格数下限属于当前玩法约束。
