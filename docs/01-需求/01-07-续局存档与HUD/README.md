# 续局存档与 HUD 需求

> 所属功能域：局内流程、HUD 与本地续局  
> 版本归属：V0.4 | 生命周期状态：已审核

## 目标

将 HUD 作为独立可复用组件，并允许玩家从暂停菜单保存退出；应用被切到后台、被系统回收或主动退出后，经典模式和挑战模式均可在下次启动时从各自最近的可恢复状态继续。

## 核心规则

- **L1-SV-01**：HUD 独立承载暂停、分数、单局正计时和挑战倒计时；在竖屏及桌面调试窗口中均不得遮挡顶部发射口、危险线或棋盘数字。
- **L1-SV-02**：经典与挑战模式各自保存一份本地续局记录；保存或恢复一方不得覆盖另一方。
- **L1-SV-03**：经典模式只保存最近一次可再次发射的稳定回合检查点。若玩家正在发射或回收中保存/被杀，恢复时回到该批发射前的分数、棋盘和球库存，不恢复飞行球。
- **L1-SV-04**：挑战模式保存当前棋盘、分数、球库存、单局正计时与下一波剩余时间；恢复时所有球都回收到顶部发射槽，棋盘保持保存时的逻辑状态，不对离线时长补算波次。
- **L1-SV-05**：暂停菜单的“保存并退出”必须保存当前模式的可恢复状态、返回模式选择页并启用该模式的“继续游戏”入口。
- **L1-SV-06**：应用进入后台、失焦或收到退出请求时自动刷新当前模式存档；终局、重新开始或主动新开局不得恢复旧的同模式存档。
- **L1-SV-07**：恢复失败、版本不兼容或数据损坏时仅丢弃对应模式存档，并保持模式选择可用；不得影响最高分记录或另一个模式存档。

## 非范围

- 不做云同步、跨设备迁移、对局回放或飞行中球的逐帧恢复。
- 不做系统级原生返回键交互设计；仅在收到引擎的退出/后台通知时执行本地持久化。
