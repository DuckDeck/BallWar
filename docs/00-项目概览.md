# Ball War 项目概览

> 引擎：Godot 4.7  
> 语言：GDScript  
> 项目形态：离线 2D 客户端

## 入口与运行

- 主场景：`res://scenes/main.tscn`
- 配置入口：`res://project.godot`
- 运行要求：使用 Godot **4.7** 导入项目后运行主场景。
- 设计基准：1080×1920 竖屏，窗口以 Canvas Items 方式等比伸缩。

## 目录职责

| 目录 | 职责 |
|---|---|
| `scenes/` | 主场景、球、几何障碍和奖励块的可组合场景。 |
| `scripts/` | 场景行为、物理、输入、程序绘制和局内流程。 |
| `resources/` | `.tres` 可调参数资源；当前 `game_config.tres` 为第一阶段统一参数源。 |
| `tests/` | 可无头运行的第一阶段回归脚本。 |
| `docs/` | 需求、交互、技术规范、测试、项目导览和任务说明。 |

## 当前已实现

- 用户可在顶部缺口下方点击，或向下拖拽后松开；球始终从顶部中央缺口向下发射。
- 活动球持续受重力影响，在左右墙、顶部斜线和当前障碍上反弹；障碍反弹使用独立的衰减系数。
- 障碍以六种随机几何形状出现：普通命中减 1 生命，归零后加 10 分；底部新行会按配置替代出 `+1` 与放大奖励格。
- `+1` 奖励命中后自然下落一颗普通球并永久增加本局库存；普通球命中放大奖励会升级为重球，重球命中则额外自然下落一颗重球。
- 活动球绘制非物理的渐隐拖尾；拖尾只记录最近运动轨迹。
- 球触及底部浅托底曲线后，停止重力和碰撞，按落点向左或向右滚出场地、在侧墙外上行、沿同侧斜顶回到顶部缺口。超时先可见下落至托底线后再回收，越过兜底线才走异常回收。

## 当前运行时结构

| 类型 | 文件 | 职责 |
|---|---|---|
| 主场景 | `scenes/main.tscn` | 组合场地边界、实体层、发射器、控制器和 HUD。 |
| 回合编排 | `scripts/game_controller.gd` | 管理模式、批次、永久球库存、得分和回合恢复。 |
| 球 | `scripts/ball.gd` | 重力、反弹、拖尾、托底回收子状态。 |
| 棋盘节点 | `scripts/obstacle.gd`、`scripts/reward_block.gd` | 几何障碍的生命/销毁，以及奖励的无反弹收集。 |
| 统一配置 | `resources/game_config.tres` | 场地、球手感、托底路线和初始障碍参数。 |
| 自动化测试 | `tests/t1_*.gd` | 冒烟、重力/拖尾、左右托底回收回归。 |

球、行为与未来扩展边界见 [项目导览](05-项目导览/README.md)。

## 顶部正计时

- 主场景启动即在右侧分隔虚线上方显示单局正计时 `Time: mm:ss`；启动初值为 `Time: 00:00`，当前在瞄准、球运动和托底回收期间持续推进。
- `GameController` 是经过时间的唯一状态所有者，并以信号通知 `Main` 更新 HUD；暂停、Game Over 与重开时的冻结/重置仍属于后续 T4。
- `tests/t1_timer_test.gd` 覆盖启动初值、65 秒累计和跨分钟格式化。

## 多球与棋盘架构

- `BallManager` 管理 `BallBatch` 内的活动球；球的定义、运行状态和拖尾颜色已拆分，球只检测世界层而不相互碰撞。
- `BoardController` 使用逻辑网格管理底部随机行、整体上移、危险线和安全回合成长；`GameController` 在最后一球回收后才提交一次波次结算。
- 具体爆炸、穿透、大体积球、更多障碍形状和结算 UI 尚未实现，但已有 `BallEffect`、`HitContext` 和 `HitResult` 扩展入口。
