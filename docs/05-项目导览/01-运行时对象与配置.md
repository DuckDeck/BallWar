# 运行时对象与配置参考

> **参考手册**：查询第一阶段对象职责、状态、信号和调参入口时使用。设计动机见 [00-design](00-design.md)，流程见 [02-单球回合与托底回收](02-单球回合与托底回收.md)。  
> 版本归属：V0.1-MVP | 生命周期状态：已审核

## 模块地图

| 模块 | 文件 | 当前职责 | 直接依赖 |
|---|---|---|---|
| 场景装配 | `scenes/main.tscn` | 静态边界、实体层、Launcher、Controller、HUD。 | `GameConfig`、Ball/Obstacle 场景。 |
| 主视图 | `scripts/main.gd` | 连接发射、分数、状态与 HUD。 | `Launcher`、`GameController`。 |
| 回合控制 | `scripts/game_controller.gd` | 单球/单障碍生成、得分、回合状态。 | Ball、Obstacle、两个 Layer。 |
| 发射器 | `scripts/launcher.gd` | 鼠标/触摸输入、瞄准线、待发球绘制。 | `GameConfig`。 |
| 球 | `scripts/ball.gd` | 活动运动、碰撞、拖尾、托底回收。 | `GameConfig`、具体 `Obstacle` 类。 |
| 障碍 | `scripts/obstacle.gd` | 固定六边形绘制、生命、销毁得分。 | `GameConfig`。 |
| 场地绘制 | `scripts/arena_renderer.gd` | 边框、顶缝、底部曲线和背景。 | `GameConfig`。 |

## 状态与信号

| 所属 | 状态/信号 | 说明 |
|---|---|---|
| `GameController` | `READY → BALL_ACTIVE → RESOLVING → READY` | 第一阶段一个回合最多只有一颗活动球。 |
| `Ball` | `ACTIVE` | 重力、`move_and_collide`、边界与障碍反弹、拖尾。 |
| `Ball` | `ROLLING → LIFTING → ROOF_RETURN` | 托底后的脚本驱动回收；不执行重力和碰撞。 |
| `Ball.recovered(reason)` | `bottom_trough`、`lifetime_expired`、`bottom_exit` | 控制器据此清理当前球并结束回合；正常路径是 `bottom_trough`。 |
| `Obstacle.damaged(remaining_health)` | 预留命中反馈 | 当前控制器不消费该信号。 |
| `Obstacle.destroyed(points)` | 障碍归零 | 控制器累加分数并在下一回合补块。 |

## 当前配置基线

`resources/game_config.tres` 是第一阶段的统一参数源；运行时不应修改共享 Resource。

| 分组 | 字段/当前值 | 含义 |
|---|---|---|
| 球 | `ball_speed=980`、`ball_gravity=4000`、`ball_radius=13.333` | 普通球的发射速度、重力和绘制/逻辑半径；球体与待发球均不绘制描边。 |
| 障碍反弹 | `obstacle_bounce_restitution=0.68` | 球命中当前六边形后保留的速度比例；越小反弹越弱。 |
| 拖尾 | `duration=0.32`、`width_multiplier=1.45` | 轨迹保留时长和球头宽度比例。 |
| 场地 | 左右墙 `72/1008`、顶端 `240`、缺口 `96` | 绘制、静态碰撞和回收路线共用的几何基线。 |
| 托底回收 | 滚动/上行/回顶 `1640/2200/1720` | 触底后的受控移动速度；不影响活动球物理。 |
| 回合 | 初始块 `(540,1040)`、生命 `3`、分数 `10` | 当前单块原型的生成数据。 |

## 已知实现边界

- `Ball._ready()` 会将 `CollisionShape2D` 同步为 `GameConfig.ball_radius`；修改该配置会一致影响绘制、碰撞、引导查询、拖尾宽度与托底偏移。
- `Ball` 通过 `if collider is Obstacle` 直接调用 `take_hit()`，目前没有携带伤害来源、命中点、法线或能力标签。
- `Obstacle` 同时绑定六边形绘制、六边形碰撞、固定扣血和固定得分；当前不能把“形状”和“行为”自由组合。
- `GameController` 保存单个 `_active_ball` 和 `_active_obstacle`，因此任何一个球回收都会结束回合。
