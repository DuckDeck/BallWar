# 单球回合与托底回收

> **理解层**：排查发射、命中、回合无法恢复或托底路径问题时阅读。对象字段见 [01-运行时对象与配置](01-运行时对象与配置.md)。  
> 版本归属：V0.1-MVP | 生命周期状态：已审核

```text
顶部缺口下方输入
  → Launcher.launch_requested(direction)
  → GameController 在 READY 创建 Ball
  → Ball.ACTIVE：重力 + 碰撞 + 拖尾
  → 命中 Obstacle：反射、速度乘 0.68、生命 -1
  → 底部浅托底曲线
  → ROLLING（按落点向左/右）
  → LIFTING（对应侧墙外上移）
  → ROOF_RETURN（侧顶 → 斜顶端 → 发球缺口）
  → Ball.recovered(bottom_trough)
  → GameController.RESOLVING → READY
```

## 活动运动

`Ball.launch()` 以归一化方向乘 `ball_speed` 设置速度。`ACTIVE` 状态的每个物理帧先叠加向下重力，再调用一次 `move_and_collide`。边界反弹保留速度大小；撞到当前障碍则按法线反射后乘 `obstacle_bounce_restitution`，并使障碍扣 1 生命。

拖尾由运行时创建的顶层 `Line2D` 保存最近位置；它只影响视觉。进入托底回收或发出回收信号时会清除，因此回顶路线没有拖尾，也不触发二次碰撞。

## 托底判定和方向

`GameConfig.get_bottom_trough_y(x)` 同时用于渲染和判定。球必须正在下落，并且球底到达曲线高度才会触发。落点在场地中心左侧向左滚，右侧向右滚；恰在中心时沿当前水平速度决定，水平速度为零则向右。

这是视觉受控路径而非“沿屋顶真实滚动”的物理模拟。它保证第一阶段可预测、可测试；若未来场地存在不同屋顶、移动墙或巨型球，应把这条路线抽到场地服务，并按球半径留出安全距离。

## 兜底与测试

正常情况下应由 `bottom_trough` 完成回合。`ball_max_lifetime` 与 `recovery_y` 只防止异常卡死，不是设计上的主要离场方式。
